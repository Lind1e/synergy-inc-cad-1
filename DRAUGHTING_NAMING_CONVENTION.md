# Draughting File Naming Convention

All draughting files, exports, research notes, and presentations must use clear names.

## Required Format

```text
project-area_document-type_version_student-or-team_date
```

## Examples

```text
synergy-zoning-plan_concept-v01_team-a_2026-01-12.pdf
synergy-workshop-layout_concept-v02_john-smith_2026-01-14.dwg
synergy-public-private-flow_review-v01_team-b_2026-01-15.png
synergy-medical-space-research_notes-v01_mary-jones_2026-01-16.md
```

## Rules

- Use lowercase letters where possible.
- Use hyphens instead of spaces.
- Include a version number.
- Include your name or team name.
- Include the date.
- Do not use file names like `final`, `latest`, `new`, or `final-final`.
- Update `REVISION_LOG.md` when making meaningful changes.
