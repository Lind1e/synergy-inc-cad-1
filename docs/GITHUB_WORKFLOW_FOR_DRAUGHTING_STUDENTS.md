# GitHub Workflow for Architectural Draughting Students

GitHub is not only for software. In this internship, you will use GitHub to practise professional version control, file organisation, issue tracking, review, and revision management.

## Basic Workflow

1. Read your assigned issue or task.
2. Work in the correct folder.
3. Name your files correctly.
4. Export review-friendly versions.
5. Update the revision log.
6. Commit and push your work.
7. Submit your work for review.

## What GitHub Helps With

- Tracking who changed what
- Keeping work organised
- Reviewing work remotely
- Keeping a project history
- Managing tasks through issues
- Recording feedback and revision actions

## Review-Friendly Files

Mentors should not need specialist software to review every item. Always export a review copy where possible.

Recommended review files:

- PDF for layouts
- PNG or JPG for quick view
- Markdown notes for explanations

## Good Commit Messages

Use messages that explain the work clearly.

Examples:

```text
Add first zoning concept for Synergy Inc
Update movement flow diagram after review
Add research notes for medical space requirements
Export parking access concept as PDF
```
